# Feature Specification: 水換えアニメーション (Water Change Animation)

**Feature Branch**: `019-water-change-animation`
**Created**: 2026-03-27
**Status**: Draft
**Input**: User description: "水換えがボタンで一瞬で終わるので、水換えモードを実装して、水槽をクリックすると水位が下がって3分の1くらいの水位まで減った後水位が元に戻るようにしたい。戻りながら水の色をブレンドして戻り切ったらその時の色になるようにしたい。また魚も水位が下がったらそれに応じて移動するようにしたい。可能ならマウスカーソルの位置にポンプを表示して画面外にホースが動的に伸びるようにしたい。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 水換えモードでの水位変化アニメーション (Priority: P1)

ユーザーが水換えボタンを押すと、水換えモード（待機状態）に入る。モード中は水槽エリアがクリック可能であることが視覚的に示される。ユーザーが水槽をクリックすると、水位がアニメーションで徐々に下がり、現在の水位の約1/3まで減少する。その後、新しい水が徐々に注がれて元の水位まで戻る。モードはアニメーション完了時に自動的に終了する。モード中にもう一度水換えボタンを押すか、水槽外をクリックするとモードがキャンセルされる。

**Why this priority**: 水換えの核心的な体験であり、この機能なしには他の視覚効果が意味をなさない。単独で水換えのリアリティを大幅に向上させる。

**Independent Test**: 水換えボタンを押して、水位が下がって戻るアニメーションが再生されることを確認するだけでテスト可能。水位変化が滑らかで自然に見えることを目視確認できる。

**Acceptance Scenarios**:

1. **Given** 水槽に水が入っている状態, **When** ユーザーが水換えボタンを押す, **Then** 水換えモード（待機状態）に入り、水槽エリアがクリック可能であることが視覚的に示される
2. **Given** 水換えモード中, **When** ユーザーが水槽エリアをクリックする, **Then** 水位がアニメーションで現在の約1/3まで徐々に下がる
2. **Given** 水位が1/3まで下がった状態, **When** 排水が完了する, **Then** 水位がアニメーションで元の水位まで徐々に戻る
3. **Given** 水換えアニメーション中, **When** アニメーションが完了する, **Then** 水の汚れ度が通常の水換えと同じだけ減少する（waterDirtiness -50, algaeLevel -10）
4. **Given** 水換えアニメーション中, **When** ユーザーがアニメーション完了前に何らかの操作を行う, **Then** アニメーションは中断されず最後まで再生される
5. **Given** 水換えモード（待機状態）中, **When** ユーザーがもう一度水換えボタンを押すか水槽外をクリックする, **Then** モードがキャンセルされ通常状態に戻る
6. **Given** 水換えアニメーションが完了する, **When** アニメーションが終了する, **Then** 水換えモードが自動的に終了し通常状態に戻る

---

### User Story 2 - 水の色ブレンドアニメーション (Priority: P1)

排水後に新しい水が注がれる際、古い水の色と新しいきれいな水の色が徐々にブレンドされる。水位が元に戻りきった時点で、水換え後の汚れ度に対応する色になっている。

**Why this priority**: 水換えの視覚的リアリティに不可欠。水位変化と同時に起こる演出であり、P1の水位アニメーションと密接に関連する。

**Independent Test**: 汚れた水（waterDirtiness が高い状態）で水換えを実行し、給水中に色がグラデーション的に変化していくことを目視確認できる。

**Acceptance Scenarios**:

1. **Given** 汚れた水の状態（例: waterDirtiness 80）, **When** 新しい水が注がれ始める, **Then** 水の色が古い色から新しい色へ徐々にブレンドされる
2. **Given** 給水アニメーション中, **When** 水位が元に戻りきる, **Then** 水の色は水換え後の汚れ度（例: waterDirtiness 30）に対応する色になっている
3. **Given** きれいな水の状態（waterDirtiness が低い）, **When** 水換えを行う, **Then** 色の変化は微小だが、アニメーション自体は同様に再生される

---

### User Story 3 - 魚の水位追従移動 (Priority: P2)

水位が下がると、魚たちも水位に合わせて下方に移動する。魚は常に水中にいるように見え、水面より上に取り残されることがない。水位が戻ると魚も元の遊泳範囲に戻る。

**Why this priority**: リアリティの向上に重要だが、水位変化アニメーション単体でも価値はある。魚が水面の上に浮いて見えると不自然なので、P2として必須に近い。

**Independent Test**: 水換え中に魚が水位に追従して移動すること、水面より上に魚が表示されないことを目視確認できる。

**Acceptance Scenarios**:

1. **Given** 水換えモードで水位が下がっている, **When** 水位が魚の現在位置より下がる, **Then** 魚は水位に合わせて下方に移動し、常に水中に留まる
2. **Given** 水位が1/3まで下がった状態, **When** 魚が泳いでいる, **Then** 魚の遊泳可能範囲が水位の範囲内に制限される
3. **Given** 水位が元に戻る過程, **When** 水位が上昇する, **Then** 魚の遊泳可能範囲も水位に応じて広がり、通常の位置に戻る

---

### User Story 4 - ポンプとホースの視覚演出 (Priority: P3 — Stretch Goal)

**注意: この User Story は stretch goal です。P1/P2 の完了で issue としては完了扱いとなります。余裕があれば実装します。**

水換え中にマウスカーソルの位置にポンプが表示され、ポンプから画面外に向かってホースが動的に伸びる。これにより、水換えの臨場感がさらに向上する。

**Why this priority**: 視覚的な付加価値であり、水換えの基本体験（水位変化・色ブレンド・魚の追従）が完成した後に取り組むべき演出機能。なくても水換えアニメーションとして成立する。issue 原文でも「可能なら」と記載されている。

**Independent Test**: 水換え中にマウスを水槽エリア上で動かし、カーソル位置にポンプアイコンが追従し、画面端に向かってホースが動的に描画されることを確認できる。

**Acceptance Scenarios**:

1. **Given** 水換えアニメーションが再生中, **When** マウスが水槽エリア上にある, **Then** マウスカーソル位置にポンプのビジュアルが表示される
2. **Given** ポンプが表示されている, **When** マウスを移動する, **Then** ポンプがカーソルに追従し、ホースが画面外（上方向）へ動的に伸びる
3. **Given** 水換えアニメーションが完了する, **When** アニメーションが終了する, **Then** ポンプとホースの表示が消える

---

### Edge Cases

- 水換えモード待機中に再度水換えボタンが押された場合：モードがキャンセルされ通常状態に戻る
- 水換えアニメーション再生中に水換えボタンが押された場合：アニメーション中はボタンを無効化する
- 水換え中にタブが非アクティブになった場合：復帰時にアニメーションの残り部分を再生する
- waterDirtiness が 0 の状態で水換えした場合：水位アニメーションは再生されるが、色の変化は実質なし
- 水換え中にエサやりボタンが押された場合：水換え完了まで他のアクションボタンも無効化する
- ウィンドウサイズが極端に小さい場合：ポンプとホースの表示は省略してよい
- 水換え中の藻オーバーレイ：水位が下がっても苔ドットは常に全範囲で表示され続ける（水位変化の影響を受けない）

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 水換えボタンを押すと水換えモード（待機状態）に入り、水槽エリアがクリック可能であることが視覚的に示される
- **FR-001a**: 水換えモード中に水槽エリアをクリックすると、水位低下アニメーションが開始される
- **FR-001b**: 水換えモード中にもう一度水換えボタンを押すか水槽外をクリックすると、モードがキャンセルされ通常状態に戻る
- **FR-001c**: 水換えアニメーション完了時にモードが自動的に終了する
- **FR-002**: 水位は元の水位の30%（最低水位比率=0.30）まで滑らかに低下する
- **FR-003**: 水位が最低点に達した後、自動的に給水アニメーションが開始され、元の水位まで戻る
- **FR-004**: 給水中、水の色がアニメーション開始時の色（現在の waterDirtiness に対応）から、waterDirtiness-50 適用後の最終色へ線形補間される
- **FR-005**: 水位が元に戻りきった時点で、水の色は waterDirtiness-50 適用後の汚れ度に対応する色に一致する
- **FR-006**: 水換え中、魚の遊泳可能範囲が現在の水位に制限される
- **FR-007**: 魚は水面より上に表示されない
- **FR-008**: 水換えアニメーション中、すべてのアクションボタン（エサやり、水換え、藻掃除、ライト）は無効化される
- **FR-009**: 水換えアニメーション完了時に、通常の水換え効果（waterDirtiness -50, algaeLevel -10）が適用される
- **FR-009a**: 水換えアニメーション中は水質関連の状態更新（waterDirtiness, algaeLevel の自動増加）を凍結し、アニメーション完了後に再開する
- **FR-010** *(stretch goal)*: 水換え中にマウスカーソルが水槽エリア上にある場合、カーソル位置にポンプビジュアルが表示される
- **FR-011** *(stretch goal)*: ポンプから画面外（上方向）に向かってホースが動的に描画される
- **FR-012** *(stretch goal)*: 水換えアニメーション完了時にポンプとホースの表示が消える
- **FR-013**: アニメーション全体の所要時間は以下の定数で定義する（後から調整可）: 排水=6.0秒、ポーズ=2.0秒、給水=6.0秒、合計=14.0秒
- **FR-014**: 排水・給水アニメーションは ease-in-out のイージングカーブを使用する（開始と終了が緩やかで中間が速い）
- **FR-015**: 排水完了後、給水開始前に2.0秒のポーズを挟み、フェーズの切り替わりを視覚的に明確にする

### Key Entities

- **WaterChangeAnimation**: 水換えアニメーションの進行状態を管理する。フェーズ（待機/排水中/ポーズ/給水中/完了）、経過時間、現在の水位割合、色ブレンド率を含む
- **PumpVisual**: マウスカーソルに追従するポンプの表示位置と、ホースの接続先座標を含む

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 水換えボタン押下から水位低下開始まで0.5秒以内にアニメーションが開始される
- **SC-002**: 排水・給水アニメーションが滑らかに再生され、目に見えるカクつきがない
- **SC-003**: 水換え完了後の水の色が、汚れ度に対応する正しい色と一致する
- **SC-004**: アニメーション中、魚が水面より上に表示されるフレームが0である
- **SC-005**: 水換えアニメーション全体が約14秒で完了する（排水6.0秒＋ポーズ2.0秒＋給水6.0秒）

## Clarifications

### Session 2026-03-27

- Q: 水換え中の藻（苔）オーバーレイの振る舞いは？ → A: 苔は常に全範囲表示のまま（水位変化に影響されない）
- Q: アニメーションのイージング（加減速）は？ → A: ease-in-out（開始と終了が緩やかで中間が速い）
- Q: 排水と給水の間のトランジションは？ → A: 短いポーズ（約0.5秒）を挟み、フェーズの切り替わりを視覚的に分かりやすくする
- Q: 水換えの開始操作フローは？ → A: ボタンでモードに入り、水槽クリックでアニメーション開始（モード制）
- Q: 色ブレンドの目標色は？ → A: 開始時の色から waterDirtiness-50 適用後の最終色へ線形補間（物理的混合ではなく演出的補間）
- Q: ポンプ・ホースは必須か任意か？ → A: P3 stretch goal。P1/P2 完了で issue 完了扱い。ポンプ・ホースは余裕があれば実装
- Q: アニメーション中の状態更新ルールは？ → A: アニメーション中は水質関連の状態更新（waterDirtiness, algaeLevel の自動増加）を凍結する
- Q: 受け入れ基準の数値定義は？ → A: 最低水位比率=0.30、排水=6.0s、ポーズ=2.0s、給水=6.0s（定数化、後から調整可）

## Assumptions

- 水位の「1/3まで下がる」は元の水位（タンク内の水の高さ）の約30%の位置まで水位が下がることを意味する
- 排水と給水のアニメーション速度は ease-in-out（開始と終了が緩やかで中間が速い）とする
- ポンプとホースのビジュアルはシンプルなアイコン/線で表現する（詳細なグラフィックスは不要）
- 水換え効果（waterDirtiness, algaeLevel の減少）はアニメーション完了時に一括で適用される（アニメーション中は見た目のみ変化）
- ホースは画面上端に向かって伸びるものとする
