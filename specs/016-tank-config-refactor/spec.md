# Feature Specification: Tank Config Refactor

**Feature Branch**: `016-tank-config-refactor`
**Created**: 2026-03-26
**Status**: Draft
**Input**: User description: "tankをfilterと同じように型を定義したい。また水槽サイズをもっと細かく分けて実寸サイズで幅と高さを持たせたい。よくある水槽サイズを参照して定義したい。また魚種によるタンク制限を魚種の最大サイズによる制限だけにして魚種のサイズからタンクに入るかどうか計算するようにして魚種から必要タンクサイズの項目を消して欲しい。またライトと水槽の間に少し幅を持たせて、ライトの光が外に拡散しているように見せたい。ライトとhudの隙間ももう少し増やしたい"

## User Scenarios & Testing

### User Story 1 - Tank Config as Typed Registry (Priority: P1)

水槽（タンク）の定義をフィルターと同じパターン（`TankConfig` インターフェース + 個別ファイル + レジストリ）に統一する。各水槽は実寸の幅・高さ（mm）を持ち、一般的なアクアリウム水槽のサイズ（20cm キューブ、30cm、45cm、60cm、90cm など）を参考にした定義とする。

**Why this priority**: 水槽の型定義は他のすべての変更（魚種制限の計算、ストアの表示、レンダリング）の基盤となるため最優先。

**Independent Test**: 新しい `TankConfig` 型で全水槽が定義され、レジストリから取得でき、既存の `TankSizeTier` enum・`TANK_BASE_CAPACITY`・`TANK_RENDER_SIZES`・`TANK_DIMENSIONS_MM` の散在した定義が一つの設定に統合されていることを確認できる。

**Acceptance Scenarios**:

1. **Given** 新しい `TankConfig` インターフェースが定義されている, **When** IDで水槽を取得する, **Then** 水槽の表示名・実寸サイズ(mm)・レンダリングサイズ・基本キャパシティ・ポモコスト・前提条件が返される
2. **Given** 水槽の実寸サイズがmmで定義されている, **When** 一般的なアクアリウム水槽サイズと比較する, **Then** 20cmキューブ(200x200mm)、30cm(300x250mm)、45cm(450x300mm)、60cm(600x360mm)、90cm(900x450mm)など実在するサイズに基づいている
3. **Given** 全水槽を取得する関数を呼び出す, **When** 結果を確認する, **Then** すべての水槽がサイズ順にリストされ、各水槽が同一の `TankConfig` 構造を持つ

---

### User Story 2 - Size-Based Fish Restriction (Priority: P2)

魚種ごとの `minTankSize` フィールドを廃止し、代わりに魚の最大サイズ（`maxSizeMm`）と水槽の実寸サイズ（幅mm）を比較して、その魚が水槽に入れるかどうかを動的に計算する。

**Why this priority**: タンク型定義（US1）に依存し、ゲームのバランスとストアの購入ロジックに直接影響するため2番目の優先度。

**Independent Test**: `minTankSize` が `GenusConfig` から削除され、代わりに魚の `maxSizeMm` と水槽の幅mmを比較する関数で購入可否が判定されることを確認できる。

**Acceptance Scenarios**:

1. **Given** ネオンテトラ（最大35mm）がいる, **When** 20cmキューブ水槽（幅200mm）への購入を試みる, **Then** 水槽幅が魚の最大サイズの一定倍以上あるため購入が許可される
2. **Given** グラミー（最大60mm）がいる, **When** 20cmキューブ水槽（幅200mm）への購入を試みる, **Then** 水槽が小さすぎるため購入が拒否される（水槽幅が魚の最大サイズの一定倍を下回る）
3. **Given** `GenusConfig` の定義を確認する, **When** `minTankSize` フィールドを探す, **Then** フィールドが存在しない

---

### User Story 3 - Light Diffusion and Spacing (Priority: P3)

ライトと水槽の間にスペースを設け、ライトの光が水槽の外側に拡散しているように見せる。また、ライトとHUD（上部オーバーレイ）の間のスペースも広げる。

**Why this priority**: ビジュアルの改善であり、機能に影響しないため最後の優先度。ただしユーザー体験を大きく向上させる。

**Independent Test**: ライトと水槽の間に視覚的なギャップがあり、光の拡散エフェクト（台形や半透明の光線）が見え、HUDとライトの間にも十分なスペースがあることを目視で確認できる。

**Acceptance Scenarios**:

1. **Given** ライトがオンの状態, **When** 水槽シーンを表示する, **Then** ライトと水槽の間にスペースがあり、光が水槽の幅より広がって拡散して見える
2. **Given** ライトがオフの状態, **When** 水槽シーンを表示する, **Then** 拡散エフェクトは表示されず、ライトと水槽の間のスペースは維持される
3. **Given** HUDが表示されている, **When** ライトの位置を確認する, **Then** HUDとライトの間に以前より広いスペースがある

---

### Edge Cases

- 既存のセーブデータが旧 `TankSizeTier` enum を使用している場合、新しい `TankId` へのマイグレーションが必要
- 魚のサイズ制限計算で境界値（ちょうど制限倍率に等しい場合）の扱い：許可とする（以上判定）
- 水槽サイズが細かく分かれることで、ストアUIにおける水槽の表示順序と購入フローが変わる可能性がある
- 全種の中で最大の `maxSizeMm` を持つ種がいない属（全種同サイズ）でも計算が正しく動作すること

## Requirements

### Functional Requirements

- **FR-001**: システムは `TankConfig` インターフェースを持ち、各水槽を個別ファイルで定義すること（フィルターと同じレジストリパターン）
- **FR-002**: `TankConfig` は ID、表示名、実寸サイズ（幅・高さ・奥行きmm）、基本キャパシティ、レンダリングサイズ、ポモコスト、前提条件、説明文を含むこと
- **FR-003**: 水槽サイズは一般的なアクアリウム水槽（20cmキューブ、30cm、45cm、60cm、90cmなど）の実寸に基づくこと
- **FR-004**: 水槽レジストリは個別取得、全件取得、ストアアイテム生成の関数を提供すること
- **FR-005**: `GenusConfig` から `minTankSize` フィールドを削除すること
- **FR-006**: 魚種が水槽に入れるかどうかは、属の中で最大の `maxSizeMm` × 4 と水槽の `widthMm` を比較して動的に計算すること（水槽幅 >= maxSizeMm × 4 で許可）
- **FR-007**: ストアの購入チェックを新しいサイズベースの判定ロジックに更新すること
- **FR-008**: ライトと水槽の間にギャップを設け、ライトの光が水槽幅より広がって拡散するビジュアルエフェクトを追加すること
- **FR-009**: ライトとHUDの間のスペースを拡大すること
- **FR-010**: 既存のセーブデータを新しい水槽IDシステムへマイグレーションすること

### Key Entities

- **TankConfig**: 水槽の完全な定義。ID、表示名、実寸サイズ（幅・高さ・奥行きmm）、基本キャパシティ、レンダリングサイズ、コスト、前提条件を持つ
- **TankId**: 水槽を一意に識別するID型（例: `'nano_20'`, `'small_30'`, `'medium_45'`, `'large_60'`, `'xl_90'`）
- **TankRegistry**: 全水槽を管理するMapベースのレジストリ（フィルターレジストリと同パターン）

## Success Criteria

### Measurable Outcomes

- **SC-001**: すべての水槽定義が単一の `TankConfig` 型で統一され、個別ファイルに定義されている
- **SC-002**: 水槽の実寸サイズが実在のアクアリウム水槽の規格に基づいている
- **SC-003**: 魚種の購入可否判定が `maxSizeMm` と水槽の `widthMm` の比較のみで行われ、ハードコードされた `minTankSize` が存在しない
- **SC-004**: 既存のセーブデータが正常にマイグレーションされ、データ損失がない
- **SC-005**: ライトの拡散エフェクトが視覚的に確認でき、ライト-水槽間およびライト-HUD間のスペースが改善されている

## Clarifications

### Session 2026-03-26

- Q: 魚の最大サイズと水槽幅の倍率は？ → A: 4倍（水槽幅 >= maxSizeMm × 4）

## Assumptions

- 水槽の幅（mm）のみで魚のサイズ制限を判定する。水深は十分にあると仮定する
- 魚の最大サイズ（`maxSizeMm`）は種（Species）レベルの値を使用する。属（Genus）の全種の中で最大のものを制限判定に使う
- 水槽幅に対する魚の最大サイズの比率は4倍（水槽幅 >= maxSizeMm × 4）。現行ゲームバランスとの整合性を検証済み
- 既存の `TankSizeTier` enum は新しい `TankId` 型に置き換えられ、マイグレーション関数で旧データを変換する
- レンダリングサイズ（ピクセル単位）は実寸サイズの比率を維持しつつ、UIに収まるようにスケーリングする
