## Original Issue Body

現状苔が下から迫り上がってくる感じだが、これを全体的にまばらな緑の点が増えていって、魚は最初は少し透過して見えるが苔が完全に水槽を覆うと魚がほとんど見えなくなる感じにしたい

## Clarification Notes

### 現状の実装
- `Tank.tsx` にて苔（algae）は水槽底部の緑色の矩形ストリップとして描画
- `algaeLevel` (0-100) に応じて高さが 0〜20px に線形スケール
- 色: `#4a8a3a`, opacity: 0.5
- 魚は苔レイヤーの上に描画されるため、苔が高くなっても魚自体の見た目には影響しない

### 変更方針
1. **描画方式の変更**: 底部の矩形ストリップ → 水槽全体にまばらに散らばる緑の点（ドット）パターン
2. **苔の進行表現**: `algaeLevel` の増加に応じてドットの密度が5段階で増加する。苔レイヤーは常に魚の前面（上）に描画する。

   | Stage | algaeLevel | 被覆率目安 | ドット opacity | 魚の視認性 |
   |-------|-----------|-----------|---------------|-----------|
   | 0 | 0 | 0% | — | 完全に見える |
   | 1 | 1-20 | ~5% | 0.3 | ほぼ完全に見える（わずかに緑の点が散在） |
   | 2 | 21-40 | ~15% | 0.4 | 魚は見えるが苔の存在が認識できる |
   | 3 | 41-60 | ~35% | 0.5 | 魚が苔越しに薄く透けて見える |
   | 4 | 61-80 | ~60% | 0.6 | 魚の輪郭がぼんやり見える程度 |
   | 5 | 81-100 | ~85% | 0.7 | 魚がほとんど見えない |

3. **苔レイヤーの適用範囲**: 苔ドットは水槽内の描画領域全体（水面〜砂の上）に配置され、魚を含む水槽内の全オブジェクトの前面に描画される。全 Stage で一貫して前面レイヤーとする。
4. **ドット増加ルール**:
   - ドットの位置は `algaeLevel` から決定論的に生成する（シード付き擬似乱数）
   - Stage 境界ではなく、`algaeLevel` の値に応じて連続的にドット数が変化する（例: level 19 → 20 → 21 でドットが1-2個ずつ増減）
   - **上昇時**: 低レベルで生成済みのドット位置は保持し、新しいドットが追加される（累積方式）
   - **減少時**（水替えによる -10 など）: 高レベルで追加されたドットから順に消える（LIFO 的）
   - `cleanAlgae` で 0 にリセットされた場合、全ドットが消える
5. **Stage 5 内の進行**: Stage 5 (81-100) 内でも被覆率は段階的に増加する。81 で約 75%、100 で約 90% を目安とし、100 でも完全不透明にはしない（わずかに魚の影が見える）

### スコープ外
- `algaeLevel` の増減ロジック（deterioration.ts）は変更しない
- 苔掃除アクション（engine.ts の cleanAlgae）のロジックは変更しない
- 健康影響の閾値（health.ts）は変更しない
- ActionBar の苔アイコンやステータスバー表示は変更しない

## Acceptance Criteria

1. **AC1**: 苔が水槽底部の矩形ではなく、水槽全面にまばらな緑のドットとして描画される
2. **AC2**: `algaeLevel` が 0 のとき、ドットは表示されない
3. **AC3**: `algaeLevel` の増加に伴い、ドットの数（密度）が連続的に増加する
4. **AC4**: 苔レイヤーは全 Stage で魚を含む水槽内全オブジェクトの前面に描画される
5. **AC5**: 5段階の視覚的な差が明確に区別でき、各 Stage の被覆率が上記テーブルの目安に概ね沿っている
6. **AC6**: 同じ `algaeLevel` に対して描画されるドットパターンが安定している（毎フレームランダムに変わらない）
7. **AC7**: `algaeLevel` 上昇時、既存ドットの位置が保持されたまま新しいドットが追加される
8. **AC8**: `algaeLevel` 減少時、高レベルで追加されたドットから順に消える
9. **AC9**: 既存のゲームロジック（deterioration, health, engine, maintenance-quality）に変更がない
10. **AC10**: `npm run lint` と `npm test` が通る
