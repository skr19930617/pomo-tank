# Feature Specification: 苔掃除アクション化

**Feature Branch**: `021-moss-cleaning-action`
**Created**: 2026-03-28
**Status**: Draft
**Input**: User description: "苔掃除がボタンを押して終わりではなく、苔掃除モードになって水槽をクリックしながらドラッグすると消しゴムっぽい挙動で擦ったエリアの苔のレベルが下がるようにしたい。実装が複雑なら単純に擦るだけで全体の苔レベルが徐々に低下して0になったら終了するようにしてもいい。苔掃除モード中はマウスカーソルがドット絵のスポンジにできるといい。"

## User Scenarios & Testing

### User Story 1 - 苔掃除モードで水槽を擦って苔を除去する (Priority: P1)

ユーザーは水槽に苔が溜まったとき、苔掃除ボタンを押して「苔掃除モード」に入る。モード中に水槽エリアをマウスでドラッグ（クリック＆ドラッグ）すると、擦った量に応じて全体の苔レベルが徐々に低下する。苔レベルが0になると自動的にモードが終了し、通常モードに戻る。

**Why this priority**: 苔掃除のインタラクティブ化が本機能の核心。ボタン一発で完了する現行動作を、ユーザーが「水槽を手入れしている」実感を得られる体験に変える。

**Independent Test**: 苔レベルが10以上の状態で苔掃除ボタンを押し、水槽上でドラッグ操作を行い、苔レベルが減少→0で自動終了することを確認できる。

**Acceptance Scenarios**:

1. **Given** 苔レベルが10以上の水槽, **When** ユーザーが苔掃除ボタンを押す, **Then** 苔掃除モードに入り、水槽エリア上でドラッグ操作が有効になる
2. **Given** 苔掃除モード中, **When** ユーザーが水槽エリアをマウスでドラッグする, **Then** ドラッグした距離・時間に応じて全体の苔レベルが徐々に低下する
3. **Given** 苔掃除モード中で苔レベルが0に達した, **When** 苔レベルが0になる, **Then** キラキラエフェクトが一瞬表示され、苔掃除モードが自動的に終了し通常モードに戻る
4. **Given** 苔掃除モード中, **When** ユーザーが苔掃除ボタンをもう一度押す, **Then** 苔掃除モードがキャンセルされ通常モードに戻る（途中の苔レベル低下は保持される）

---

### User Story 2 - スポンジカーソル表示 (Priority: P2)

苔掃除モードに入ると、マウスカーソルがドット絵のスポンジに変わり、ユーザーに「掃除中」であることが視覚的に伝わる。モード終了時に通常カーソルに戻る。

**Why this priority**: 視覚フィードバックはユーザー体験を大きく向上させるが、機能としてはP1が動作すればカーソル変更なしでも最低限成立する。

**Independent Test**: 苔掃除モード突入時にカーソルがスポンジに変わり、終了時にデフォルトに戻ることを目視で確認できる。

**Acceptance Scenarios**:

1. **Given** 通常モード, **When** 苔掃除モードに入る, **Then** マウスカーソルがドット絵のスポンジアイコンに変わる
2. **Given** 苔掃除モード中, **When** モードが終了する（苔レベル0到達またはキャンセル）, **Then** マウスカーソルが通常のカーソルに戻る

---

### User Story 3 - 擦り操作の視覚フィードバック (Priority: P3)

苔掃除モード中にドラッグすると、擦っている箇所付近に泡や拭き取りエフェクトが表示され、掃除している実感が得られる。

**Why this priority**: 操作感を高める演出だが、P1・P2が揃えば機能としては成立する。

**Independent Test**: ドラッグ中に擦り箇所付近にエフェクトが表示されることを目視で確認できる。

**Acceptance Scenarios**:

1. **Given** 苔掃除モード中, **When** ユーザーが水槽エリアをドラッグする, **Then** ドラッグ位置周辺に泡のエフェクトが表示される
2. **Given** 苔掃除モード中, **When** ユーザーがドラッグを停止する, **Then** エフェクトが自然にフェードアウトする

---

### Edge Cases

- 苔レベルが10未満のとき苔掃除ボタンは無効（既存動作を維持）
- 苔掃除モード中に水換えモードに切り替えようとした場合、苔掃除モードを先に終了してから水換えモードに入る（モードの排他制御）
- 苔掃除モード中にポモドーロタイマーが終了しても、モードは維持される（ユーザー操作を中断しない）
- ドラッグせずにクリックだけした場合も微量の苔レベル低下が発生する
- 苔掃除モード中にウィンドウからマウスが外れた場合、ドラッグ状態をリセットする
- 苔掃除モード中は魚クリック・餌やり等すべてのタンク内インタラクションを無効にする
- ドラッグは水槽エリア内で開始した場合のみ有効とし、水槽外に出たらドラッグ状態をリセットする

## Requirements

### Functional Requirements

- **FR-001**: システムは苔掃除ボタン押下で「苔掃除モード」に遷移しなければならない
- **FR-002**: 苔掃除モード中、水槽エリア上でのマウスドラッグ操作に応じて全体の苔レベルを徐々に低下させなければならない
- **FR-003**: 苔レベルの低下量は主にドラッグの移動距離に比例し、ドラッグ中は時間経過でも微量低下しなければならない。クリックのみの場合も固定微量の低下が発生する
- **FR-004**: 苔レベルが0に達したとき、キラキラエフェクトを一瞬表示してから苔掃除モードを自動終了しなければならない
- **FR-005**: 苔掃除モード中にもう一度苔掃除ボタンを押すとモードをキャンセルし通常モードに戻らなければならない
- **FR-011**: 苔掃除モード中、ActionBarの苔掃除ボタンはアクティブ色（明るい緑）でハイライト表示しなければならない
- **FR-006** (fast-follow/P2): 苔掃除モード中はマウスカーソルをドット絵スポンジアイコンに変更しなければならない
- **FR-007** (fast-follow/P2): モード終了時にカーソルを通常に戻さなければならない
- **FR-008**: 苔掃除モードと水換えモードは排他的でなければならない（同時に有効にならない）
- **FR-009** (fast-follow/P3): 苔掃除モード中のドラッグ操作時に、ドラッグ位置付近に泡エフェクトを表示しなければならない
- **FR-010**: 苔レベルが10未満の場合、苔掃除モードに入れないようにしなければならない（既存動作を維持）

### Key Entities

- **苔掃除モード状態**: モードのON/OFF、現在のドラッグ状態（ドラッグ中か否か）、累計ドラッグ距離
- **スポンジカーソル**: ドット絵で表現されたスポンジのカスタムカーソル画像

## Success Criteria

### Measurable Outcomes

- **SC-001**: ユーザーが苔掃除ボタンを押してから苔掃除モードに遷移するまで0.5秒以内
- **SC-002**: ドラッグ操作中の苔レベル低下がリアルタイムに視覚反映される（60fps描画を維持）
- **SC-003**: 苔レベル100の状態から0にするまでに必要なドラッグ操作が約5秒（チューニング目安、非厳密。距離ベース＋時間補正モデル）
- **SC-004**: 苔掃除モード中のカーソル変更が即座に反映される
- **SC-005**: モード排他制御が正しく動作し、苔掃除中に水換えモードが同時有効にならない

## Clarifications

### Session 2026-03-28

- Q: 苔レベル100→0にするのに必要な操作時間の目安は？ → A: 中程度（5秒前後）— しっかり擦る感覚
- Q: 苔掃除モード中のActionBarボタンの見た目は？ → A: アクティブ色（明るい緑）でハイライトしモード中を示す
- Q: 苔掃除完了時（苔レベル0到達）のフィードバックは？ → A: キラキラ/ピカピカのエフェクトを一瞬表示して完了を演出
- Q: 苔レベル低下の計算モデルは？ → A: 距離ベース＋時間補正。基本はドラッグ移動距離に比例して低下し、ドラッグ中は時間経過でも微量低下する。5秒(苔100→0)はチューニング目安（非厳密）。クリックのみは固定微量低下。
- Q: MVPスコープは？ → A: P1（ドラッグ苔掃除）のみMVP必須。P2（スポンジカーソル）・P3（泡エフェクト）はfast-follow（同ブランチで実装するが未実装でもマージ可）。
- Q: 苔掃除モード中の他タンクインタラクションは？ → A: 全インタラクション無効。魚クリック・餌やり等すべて無効。ドラッグは水槽内開始のみ有効、水槽外に出たらドラッグ状態リセット。
- Q: プラットフォーム対応は？ → A: デスクトップ（マウス）限定。タッチデバイスは対象外。
- Q: 苔レベルの内部精度は？ → A: 内部はfloat管理、UI表示はMath.ceilで整数表示。閾値判定（苔≥10でボタン有効）はfloat値で評価。

## Assumptions

- 苔レベルの低下は「擦った箇所の局所的な除去」ではなく「全体の苔レベルの均一な低下」とする（issueの簡易実装案を採用）
- スポンジのドット絵は既存のアイコンスタイル（8×8ピクセルビットマップ）に合わせる
- 泡エフェクトは軽量な描画で実現し、パフォーマンスに影響を与えない
- 本機能はデスクトップ（マウス操作）限定とし、タッチデバイスは対象外とする
- 苔レベルは内部的にfloatで管理し、UI表示はMath.ceilで整数に丸める。閾値判定（苔≥10でボタン有効化）はfloat値で評価する
