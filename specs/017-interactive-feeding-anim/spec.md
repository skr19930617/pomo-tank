# Feature Specification: インタラクティブ餌やりアニメーション

**Feature Branch**: `017-interactive-feeding-anim`
**Created**: 2026-03-26
**Status**: Draft
**Input**: User description: "メンテナンスの餌やりボタンを押した後、カーソルが餌アイコンに変化し、水槽内クリックで缶傾きアニメーション→粉状の餌が落下→魚が餌に集まる→群れに復帰する一連のインタラクティブ体験"

## Clarifications

### Session 2026-03-26

- Q: 餌投下のクリック可能エリアは水面付近のみか、水中全体か？ → A: 水中全体をクリック可能。餌粒はクリックしたX座標の水面位置から落下する
- Q: アニメーション完了後に再度餌やりできるか？ → A: アニメーション完了後、ボタン有効条件（hunger > 閾値）を満たしていれば何度でも餌やり可能
- Q: 魚の性格（personality）により餌への反応に差をつけるか？ → A: 性格により反応速度に差をつける（active→即反応、social→やや遅延、calm→遅延、timid→最も遅延）

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 餌やりモードの開始とカーソル変化 (Priority: P1)

ユーザーが餌やりボタンを押すと、即座に餌やりが完了するのではなく「餌やりモード」に入る。マウスカーソルが餌缶（または餌粒）のアイコンに変化し、水槽内のどこに餌を落とすかをユーザーが選べる状態になる。

**Why this priority**: 餌やりモードへの遷移がなければ後続のすべてのインタラクション（クリック位置選択、アニメーション、魚の集合）が成立しない。この機能単体でも「ボタン即完了」から「ユーザー参加型」への体験向上を実現できる。

**Independent Test**: 餌やりボタンを押してカーソルが変化することを確認し、水槽外クリックまたはESCキーでモードをキャンセルできることを検証する。

**Acceptance Scenarios**:

1. **Given** 水槽が表示されていて餌やりボタンが有効な状態, **When** ユーザーが餌やりボタンをクリック, **Then** カーソルが餌アイコンに変化し、水槽エリアがクリック可能な「餌やりモード」になる
2. **Given** 餌やりモード中, **When** ユーザーが水槽外をクリックまたはESCキーを押す, **Then** 餌やりモードがキャンセルされ、カーソルが通常に戻り、餌は消費されない
3. **Given** 餌やりモード中, **When** ユーザーが水槽内の水中エリア任意の位置をクリック, **Then** クリックしたX座標の水面位置が餌の投下地点として確定し、次のアニメーションフェーズに進む

---

### User Story 2 - 缶傾きアニメーションと餌粒落下 (Priority: P1)

水槽内クリック後、クリック位置の上部付近に餌缶が表示され傾くアニメーションが再生される。缶が傾くと同時に複数の粉状の餌粒がクリック位置から水中をゆっくり落下し、徐々にフェードアウトして消える。

**Why this priority**: 餌やりの視覚的な満足感の核心部分。餌粒の落下がなければ魚の集合行動のターゲットも存在しない。

**Independent Test**: 餌やりモードで水槽内をクリックし、缶アニメーションと餌粒の落下・消滅を目視で確認する。

**Acceptance Scenarios**:

1. **Given** 餌やりモードで水槽内をクリック, **When** クリックが確定, **Then** クリック位置上部に餌缶が表示され、傾くアニメーションが再生される
2. **Given** 缶が傾いた状態, **When** アニメーションが進行, **Then** 複数の小さな粉状の餌粒がクリック位置から水中に散布され、ゆっくり落下する
3. **Given** 餌粒が落下中, **When** 一定時間が経過, **Then** 餌粒は徐々に透明度が下がりフェードアウトして消滅する
4. **Given** 缶アニメーション完了後, **When** 餌粒がすべて消滅, **Then** 餌缶の表示が消え、餌やりモードが終了し、実際の餌やり効果（hunger減少）が適用される

---

### User Story 3 - 魚が餌に集まる行動 (Priority: P2)

餌粒が落下し始めると、水槽内の魚たちが餌の位置に向かって泳ぎ始める。群れ（スクーリング）で行動している魚は個体が群れから離れて餌に向かい、餌が消滅した後に元の群れ行動に復帰する。

**Why this priority**: 餌やり体験のリアリティと楽しさを大幅に向上させるが、Story 1・2のアニメーション基盤が前提となるためP2とする。

**Independent Test**: 餌粒が落下中に各魚の移動方向が餌位置に向かっていることを確認し、餌消滅後に通常の遊泳パターン（群れ行動含む）に復帰することを検証する。

**Acceptance Scenarios**:

1. **Given** 餌粒が水槽内に存在する状態, **When** 魚が餌粒を検知, **Then** 魚が性格に応じた反応速度（active→即、social→やや遅延、calm→遅延、timid→最も遅延）で餌粒の位置に向かって泳ぎ始める
2. **Given** 群れ行動中の魚（例: ネオンテトラ）, **When** 餌粒が出現, **Then** 個体が群れから離れて餌に向かう（群れの結束力より餌の引力が優先される）
3. **Given** 魚が餌の周辺に集まっている状態, **When** 餌粒がすべて消滅, **Then** 魚が通常の遊泳パターンに徐々に戻り、群れの魚は再び群れに合流する
4. **Given** 魚の泳層（upper/middle/lower）が異なる種がいる場合, **When** 餌が投下される, **Then** 各魚は自分の泳層の範囲内で餌に最も近い位置を目指す

---

### Edge Cases

- 餌やりモード中にタイマーが終了した場合 → 餌やりモードを自動キャンセルし、通常に戻る
- 餌粒落下中に別のアクション（水替え等）ボタンが押された場合 → 餌やりアニメーション中は他のメンテナンスボタンを無効化する
- 死んだ魚は餌に反応するか → 死んだ魚は餌に反応せず、現在位置に留まる
- 照明がオフの状態で餌やりボタンが押された場合 → 既存の仕様通り照明オフ時は餌やりボタンが無効化される
- 水槽内に魚がいない状態で餌やりした場合 → 缶・餌粒アニメーションは通常通り再生するが、集合行動は発生しない
- 餌やりモード中にウィンドウがフォーカスを失った場合 → モードを維持し、フォーカス復帰後も継続する
- 連続で餌やりした場合 → 前回アニメーション完了後であれば、hunger閾値条件を満たす限り何度でも餌やり可能

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 餌やりボタン押下時、即座に効果を適用せず「餌やりモード」に遷移しなければならない
- **FR-002**: 餌やりモード中、マウスカーソルが餌関連のカスタムアイコンに変化しなければならない
- **FR-003**: 餌やりモード中、水槽の水中エリア全体でのクリックを受け付け、クリックしたX座標の水面位置を餌投下地点としなければならない
- **FR-004**: 水槽外クリックまたはESCキーにより餌やりモードをキャンセルできなければならない
- **FR-005**: クリック後、餌缶がクリック位置上部に表示され傾くアニメーションを再生しなければならない
- **FR-006**: 缶アニメーションに連動して、クリック位置から複数の粉状餌粒パーティクルを生成しなければならない
- **FR-007**: 餌粒は水中をゆっくり落下し、一定時間後にフェードアウトして消滅しなければならない
- **FR-008**: 餌粒が存在する間、生きている魚は性格に応じた反応速度で餌の位置に向かって泳ぐ行動を取らなければならない（active→即反応、social→やや遅延、calm→遅延、timid→最も遅延）
- **FR-009**: 群れ行動中の魚は個体が群れから離脱して餌に向かい、餌消滅後に群れに再合流しなければならない
- **FR-010**: 魚は各自の泳層制約の範囲内で餌に最も近い位置を目指さなければならない
- **FR-011**: 餌やりの効果（hunger減少）は、アニメーション完了後に適用しなければならない
- **FR-012**: 餌やりアニメーション中は他のメンテナンスボタンを無効化しなければならない
- **FR-013**: 餌やりアニメーション完了後、ボタン有効条件（hunger > 閾値）を満たしていれば再度餌やりモードに入れなければならない

### Key Entities

- **餌やりモード（Feeding Mode）**: 餌ボタン押下からアニメーション完了までの一連の状態を管理するモード。アクティブ/非アクティブの状態を持つ
- **餌粒パーティクル（Food Particle）**: 投下位置、落下速度、透明度、生存時間を持つ個々の餌粒。複数が同時に存在する
- **餌缶（Food Can）**: 表示位置と傾き角度を持つアニメーション要素。クリック位置上部に出現し、アニメーション後に消滅する
- **集合ターゲット（Attraction Target）**: 餌粒群の重心位置。魚の移動先として使用される。餌粒消滅と共に無効化される

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 餌やりボタン押下からカーソル変化までの遅延がユーザーにとって知覚できないレベルであること
- **SC-002**: 缶傾きアニメーションと餌粒落下が滑らかに再生され、視覚的なカクつきがないこと
- **SC-003**: 餌粒が出現してから消滅するまでの全行程が3〜5秒の間で完了すること
- **SC-004**: 水槽内の生きている魚の80%以上が、餌粒出現後2秒以内に餌方向への移動を開始すること
- **SC-005**: 群れ行動中の魚が餌消滅後5秒以内に通常の群れ行動パターンに復帰すること
- **SC-006**: 餌やり体験全体（ボタン押下→アニメーション完了）が10秒以内で完了すること

## Assumptions

- カスタムカーソルはWebview内で対応可能な方法を用いる想定
- 餌粒パーティクルの数は5〜10個程度を想定し、パフォーマンスに大きな影響を与えない
- 餌缶のビジュアルアセットは新規に作成する必要がある（既存アセットにはない）
- 魚の泳層制約は既存のswimLayer仕様を維持し、餌のために泳層を超えることはない
- 餌やり効果（hunger減少量）は既存仕様を維持する
