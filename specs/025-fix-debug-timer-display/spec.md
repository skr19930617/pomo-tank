# Feature Specification: デバッグ加速時のタイマー表示修正

**Feature Branch**: `025-fix-debug-timer-display`
**Created**: 2026-03-29
**Status**: Draft
**Input**: User description: "デバッグで加速すると時刻表示がおかしい — 50倍にすると秒だけ正しそうだが分の表示が0に戻ったりする"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - デバッグ加速時にタイマーが正確に表示される (Priority: P1)

開発者がデバッグパネルで速度倍率（5x, 10x, 50x）を設定してポモドーロタイマーを動作させる際、タイマー表示（MM:SS または HH:MM:SS）が加速後の経過時間を正しく反映し続ける。分表示が不正に0にリセットされたり、秒と分の間で不整合が発生したりしない。

**Why this priority**: タイマー表示はポモドーロアプリのコア機能であり、デバッグ時の動作確認に不可欠。表示が不正だと開発者がゲームロジックの正当性を検証できない。

**Independent Test**: デバッグパネルで50x速度を設定し、タイマーが00:00から連続的に増加することを目視確認する。特に59秒→1:00、59:59→1:00:00の境界で分が正しく繰り上がることを確認する。

**Acceptance Scenarios**:

1. **Given** フォーカスモードでタイマーが動作中, **When** デバッグパネルで速度を50xに設定する, **Then** タイマーの分表示が正しく繰り上がり続ける（00:59 → 01:00 → 01:01 ...）
2. **Given** 50x速度でタイマーが動作中, **When** タイマーが59:59を超える, **Then** 表示が1:00:00に正しく切り替わり、分表示が00から再び増加する
3. **Given** 50x速度でタイマーが動作中, **When** 速度を1xに戻す, **Then** タイマーが現在の表示値から通常速度で継続する（値がジャンプしない）

---

### User Story 2 - ブレイクモードのカウントダウンが加速時に正確に動作する (Priority: P2)

ブレイクモードでカウントダウンタイマーを加速した際にも、残り時間が正しく減少し、分表示が不正な挙動をしない。

**Why this priority**: ブレイクモードも同じタイマーロジックを共有しており、同じバグが存在する可能性がある。

**Independent Test**: ブレイクモードに入り、50x速度でカウントダウンが05:00から正しく減少することを確認する。

**Acceptance Scenarios**:

1. **Given** ブレイクモードでカウントダウンが動作中, **When** 速度を50xに設定する, **Then** 残り時間の分表示が正しく減少する（05:00 → 04:59 → ... → 00:00）
2. **Given** 50x速度でブレイクカウントダウンが動作中, **When** 残り時間が00:00に達する, **Then** タイマーが0で停止し、負の値にならない

---

### Edge Cases

- 速度倍率を変更した直後にタイマー値が大幅にジャンプしないか？
- サーバーティック（状態同期）の間隔が短くなった際に、クライアント補間との間で不整合が発生しないか？
- 長時間（数時間分の加速）動作させた場合に、数値のオーバーフローや精度劣化が発生しないか？
- 加速中にポーズ→再開した場合に、タイマー表示が正しく継続するか？

## Assumptions

- **時間の権威ソース**: 表示時間はサーバーが提供する `timeSinceLastMaintenance`（フォーカス）/ `breakRemainingMs`（ブレイク）を基準値とし、クライアントが `(Date.now() - base.clientTs) * tickMultiplier` で補間した値を表示する。サーバー値が更新された時点でクライアント基準値がリセットされる。
- **秒の丸めルール**: フォーカスモード（カウントアップ）は `floor(totalMs / 1000)` で切り捨て、ブレイクモード（カウントダウン）も `floor(remainingMs / 1000)` で切り捨て（0未満にはならない）。
- **表示の連続性**: 高倍率（50x等）では実際の描画フレーム間に複数秒が経過するため、すべての中間秒が画面に表示されるとは限らない。正確性の基準は「各描画フレームで表示される値が、その時点の経過時間から正しく計算されていること」および「表示値が単調増加（フォーカス）/ 単調減少（ブレイク）であること」とする。
- **分=00 の正当性**: 分表示が00になるのは、時間コンポーネントが繰り上がり HH:MM:SS 形式に切り替わった場合のみ正当。MM:SS 形式のまま分が00に戻る現象がバグである。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: フォーカスモードのカウントアップタイマーは、デバッグ速度プリセット（1x, 5x, 10x, 50x）でMM:SS形式の分・秒が正確に表示されなければならない。各描画フレームの表示値は `floor((serverMs + clientElapsed * multiplier) / 1000)` から導出された値と一致すること。
- **FR-002**: ブレイクモードのカウントダウンタイマーは、デバッグ速度プリセットで残り時間が正確に減少表示されなければならない。表示値は `max(0, floor((serverMs - clientElapsed * multiplier) / 1000))` と一致すること。
- **FR-003**: タイマーが59:59を超えた場合、HH:MM:SS形式に正しく切り替わらなければならない。分=00 は HH:MM:SS 形式でのみ許容される。
- **FR-004**: 速度倍率を変更した際に、変更直前の表示値と変更直後の表示値の差分が1秒（表示秒数）以内でなければならない。基準は「倍率変更イベント発生時点の表示秒数」で計測する。
- **FR-005**: サーバーティック受信時に、クライアント補間値がサーバー提供値にリセットされ、以降はリセット後の値を基準に再補間しなければならない。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 速度50xでフォーカスタイマーを5分間（実時間6秒）動作させた際に、分表示がMM:SS形式のまま0にリセットされることなく正しく増加する
- **SC-002**: 全ての速度プリセット（1x, 5x, 10x, 50x）で、タイマー表示の分→秒の境界（XX:59→XX+1:00）が正しく遷移する
- **SC-003**: 速度倍率を変更した直前・直後の表示秒数の差分が1秒以内に収まる
- **SC-004**: ポーズ→再開後にタイマー表示がポーズ時点の値から継続し、不正なジャンプが発生しない
