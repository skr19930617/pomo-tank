# Feature Specification: 水槽の成長システムとライトスイッチ機能

**Feature Branch**: `002-tank-growth-light`
**Created**: 2026-03-24
**Status**: Draft
**Input**: User description: "水槽の大きさが最初から画面いっぱいになっているのをサイズを解放するたびに大きくしていきたい。また水槽は机の上に乗っていることがわかるようなデザインにしたい。水槽の上に歯槽のライトがあってこれをオンオフできるようなスイッチを表現してスイッチをオフにすると水槽内が暗くなってタイマーを止めるようにしたい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 水槽サイズが段階的に成長する (Priority: P1)

ユーザーがポモポイントを貯めてストアで水槽サイズをアップグレードすると、水槽の表示サイズが視覚的に大きくなる。最初のNanoタンクは小さく表示され、アップグレードするたびに画面内で水槽が大きくなっていくことで、成長の達成感を得られる。

**Why this priority**: 現在は水槽サイズをアップグレードしても魚の収容数が増えるだけで、見た目の変化がない。視覚的な成長フィードバックはユーザーのモチベーション維持に最も重要な要素である。

**Independent Test**: Nanoタンクから開始し、各サイズへアップグレードした際に水槽の描画サイズが段階的に大きくなることを目視確認できる。

**Acceptance Scenarios**:

1. **Given** 新規ユーザーがNanoタンクで開始した状態, **When** 水槽パネルを開く, **Then** 水槽は画面の最小サイズ（パネル幅の約50%程度）で表示される
2. **Given** Nanoタンクを所有している状態, **When** ストアでSmallタンクにアップグレードする, **Then** 水槽の描画サイズが一段階大きくなる
3. **Given** 任意のサイズの水槽を所有している状態, **When** 次のサイズにアップグレードする, **Then** 水槽サイズが段階的に拡大し、最大サイズ（XL）ではパネル幅いっぱいに表示される
4. **Given** アップグレード後の状態, **When** パネルを再度開く, **Then** アップグレード後のサイズが保持されている

---

### User Story 2 - 水槽が机の上に置かれているデザイン (Priority: P2)

水槽の下に机の天板が描画され、水槽が机の上に載っている見た目になる。これにより、水槽がインテリアの一部として存在感を持ち、より愛着が湧くデザインになる。

**Why this priority**: 没入感とビジュアルの質を高める重要な要素だが、機能面への影響はないためP2とする。

**Independent Test**: 水槽パネルを開いた際に、水槽の下に机の天板が描画されていることを目視確認できる。

**Acceptance Scenarios**:

1. **Given** 水槽パネルを開いた状態, **When** 水槽を表示する, **Then** 水槽の下部に机の天板が描画され、水槽が机の上に載っているように見える
2. **Given** 水槽サイズがアップグレードされた状態, **When** 水槽が大きくなる, **Then** 机のデザインも水槽サイズに合わせて調整される
3. **Given** 異なるVSCodeテーマ（ライト/ダーク）を使用している状態, **When** 水槽パネルを表示する, **Then** 机のデザインがテーマに馴染む配色で表示される

---

### User Story 3 - ライトスイッチで水槽の明暗とタイマーを制御する (Priority: P1)

水槽の上部にアクアリウムライトが描画され、ユーザーがクリックでオン・オフを切り替えられる。ライトをオフにすると水槽内が暗くなり、ポモドーロタイマー（劣化タイマー）が停止する。ライトをオンにするとタイマーが再開する。これにより、休憩や離席時にタイマーを直感的に制御できる。

**Why this priority**: ライトスイッチによるタイマー制御は、ユーザーが離席時に魚の健康が悪化するのを防ぐ重要な機能であり、ゲーム体験の核心に関わるためP1とする。

**Independent Test**: ライトスイッチをオフにして水槽が暗くなり劣化が停止すること、オンに戻して復帰することを確認できる。

**Acceptance Scenarios**:

1. **Given** 水槽パネルが表示されている状態, **When** 水槽上部のライトエリアを見る, **Then** アクアリウムライトとオン・オフスイッチが表示されている
2. **Given** ライトがオンの状態, **When** ライトスイッチをクリックする, **Then** ライトがオフになり、水槽内が暗い表現（暗めのオーバーレイ）に変わる
3. **Given** ライトがオフの状態, **When** 劣化タイマーの進行を確認する, **Then** 魚の空腹・水の汚れ・藻の成長が停止している
4. **Given** ライトがオフの状態, **When** ライトスイッチを再度クリックする, **Then** ライトがオンに戻り、水槽が通常の明るさになり、劣化タイマーが再開する
5. **Given** ライトがオフの状態, **When** ステータスバーのタイマー表示を確認する, **Then** タイマーが一時停止していることがわかる表示になっている

---

### Edge Cases

- 水槽サイズが最大（XL）の状態でさらにアップグレードしようとした場合 → ストアに次のサイズが表示されない（既存動作を維持）
- ライトオフ中にメンテナンスアクション（餌やり・水替え・藻掃除）を実行しようとした場合 → ライトオフ中もメンテナンスアクションは実行可能とする（ユーザーの利便性優先）
- VSCodeを閉じて再度開いた際のライトの状態 → ライトオンの状態で復帰する（セッション開始時は常にアクティブ）
- 水槽サイズ変更時の魚のアニメーション範囲 → 水槽の描画サイズに応じて魚の移動範囲も自動調整される
- サイドバーのコンパニオンビューでのライト表示 → ライトの明暗状態は反映するが、スイッチ操作はメインパネルでのみ可能

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 水槽の描画サイズはタンクサイズ（Nano/Small/Medium/Large/XL）に連動して段階的に変化しなければならない
- **FR-002**: Nanoタンクは最小の描画サイズで表示され、XLタンクはパネル幅いっぱいに表示されなければならない
- **FR-003**: 水槽の下部に机の天板が描画され、水槽が机上に設置されているように見えなければならない
- **FR-004**: 机のデザインは水槽サイズの変化に追従して調整されなければならない
- **FR-005**: 水槽上部にアクアリウムライトが描画されなければならない
- **FR-006**: ライトにはクリック可能なオン・オフスイッチが表示されなければならない
- **FR-007**: ライトオフ時、水槽内に暗さを表現する視覚的エフェクトが適用されなければならない
- **FR-013**: ライトオフ時、魚の移動速度が通常の50%に低下し、休眠的な表現となる
- **FR-008**: ライトオフ時、すべての劣化処理（空腹増加・水汚れ・藻の成長）が停止しなければならない
- **FR-009**: ライトオン時、劣化処理が再開しなければならない
- **FR-012**: ライトオフ時、メンテナンス間隔タイマー（ポモタイミングボーナス計算用）も一時停止し、ライトオン復帰後はオフ前の経過時間から再開しなければならない
- **FR-010**: ライトのオン・オフ状態がステータスバーに反映されなければならない
- **FR-011**: 魚の移動可能範囲は水槽の描画サイズに応じて自動調整されなければならない

### Key Entities

- **水槽サイズ段階**: 各タンクサイズ（Nano〜XL）に対応する描画寸法の定義。タンクサイズと描画サイズのマッピング。
- **ライト状態**: オン・オフの二値状態。劣化タイマーの有効・無効と連動。セッション開始時はオンがデフォルト。
- **机**: 水槽の下に描画されるビジュアル要素。水槽サイズに追従するサイズ。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーが水槽サイズをアップグレードした際、視覚的なサイズ変化を100%の確率で認識できる（各段階で明確なサイズ差がある）
- **SC-002**: 初回起動時の水槽が「小さい水槽」として認識され、最大サイズとの視覚的な差異が明確である
- **SC-003**: ライトスイッチの操作は1クリックで完了し、即座に視覚フィードバック（明暗変化）が表示される
- **SC-004**: ライトオフ中に魚の健康状態が悪化しない（劣化が完全に停止している）
- **SC-005**: ユーザーが水槽を見た際に「机の上に置かれている」と自然に認識できるデザインである
- **SC-006**: ライトオフからオンへの復帰時、劣化タイマーがオフ直前の状態から正確に再開する

## Clarifications

### Session 2026-03-24

- Q: ライトオフ中のメンテナンス間隔タイマー（ポモタイミングボーナス計算用）の扱いは？ → A: メンテナンス間隔タイマーも一時停止する（復帰後、オフ前の経過時間から再開）
- Q: ライトオフ中の魚のアニメーションはどうなるか？ → A: 魚はゆっくり泳ぐ（速度50%低下、休眠的な表現）

## Assumptions

- 水槽サイズの段階的描画は、既存のタンクサイズカタログ（Nano/Small/Medium/Large/XL）の5段階をそのまま利用する
- 机のデザインはピクセルアート調で、既存の水槽ビジュアルスタイルと統一する
- ライトのデザインもピクセルアート調で、水槽上部に横長のバー型ライトとして表現する
- ライトオフ時の暗さ表現は半透明の暗いオーバーレイで実現する
- ライト状態は永続化せず、VSCode起動時は常にオン状態で開始する
- コンパニオンビュー（サイドバー）ではライトの明暗状態は反映するが、スイッチUI自体は表示しない
